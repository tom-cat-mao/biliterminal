# BiliTerminal 原版 Python → TS 主线对照表

本文档用于回答两个问题：

1. 原版 Python 版本到底提供了哪些视频主线能力与接口；
2. 这些能力现在分别由哪些 TypeScript 模块承接，是否已经迁移完成。

对照基线：

- 原版归档：`legacy/python/bili_terminal/bilibili_cli.py`
- 当前主线：`src/**`

结论先行：

- **视频主线 CLI / REPL / TUI / API / 存储 / 路径适配 已全部切到 TS 主线**
- **Python 版本已整体归档到 `legacy/python/`，不再参与主运行链**
- **README 截图生成脚本仍保留为 Python 工具，但已经归档，不影响主功能**

### 当前已经确认的 TUI 对齐点

- `d`：默认搜索词为空时，会即时拉取 `searchDefault()` 后再搜索；
- `r`：刷新当前列表时会强制重新拉评论；
- `history` / `favorites` 模式下分页按键会给出“当前列表没有分页”的提示；
- `favorites` 视图与详情 / 评论面板的切换逻辑已由 `test/tui/App.test.tsx` 覆盖。

## 7. 核对结论

### 已完成的“原版 → TS”迁移范围

- CLI 子命令：**已完成**
- REPL 交互命令：**已完成**
- TUI 视频主线键位：**已完成**
- API 访问与评论链路：**已完成**
- 历史 / 收藏 / 搜索词存储：**已完成**
- 跨平台状态目录：**已完成，并且较原版增强**

### 当前不再保留为主运行链的部分

- Python 运行时 fallback：**已移除**
- Python 打包 payload：**已移除**
- Python 主测试链：**已移除**

### 验证锚点

- 原版归档基线：`pnpm run legacy:python-baseline`
- TS 主线测试：
  - `test/core/*.test.ts`
  - `test/api/bilibili-client.test.ts`
  - `test/storage/history-store.test.ts`
  - `test/cli/*.test.ts`
  - `test/tui/*.test.tsx`
- 主线验收：
  - `pnpm run ci`
  - `pnpm run smoke`
  - `pnpm run package:macos`

如果后续还需要继续补“逐函数级别”的 Python/TS 对照，可以在本文件基础上继续向下细化；但从**用户可见功能、命令接口、模块职责与测试锚点**这四层看，当前已经足够支撑“原版能力是否已完成迁移”的判断。
